Building the Mathematics Department Website

1. Introduction

Our team embarked on creating a comprehensive and visually appealing website for the Mathematics Department of NUTECH. The website aims to provide detailed information about the department, faculty, courses, and other relevant resources. The website's design emphasizes user-friendly navigation and accessibility, ensuring that students, faculty, and other visitors can easily find the information they need.

Purpose and Objectives of the Website:

  • Centralize information about the Mathematics Department.
  • Facilitate easy access to faculty profiles and course details.
  • Enhance global accessibility and engagement among students and faculty.
  • Support efficient organization and management of departmental resources.

2. Description of Website Functionalities and Features:

The Mathematics Department website includes:

  • Centralized information hub for department news, events, and resources.
  • Comprehensive faculty profiles with detailed information and contact links.
  • Detailed course information, including syllabi and descriptions.
  • Interactive features like an event calendar and contact forms.
  • Social media integration and accessibility features.
  • Target audience and user requirements.

3. Technology Stack

  • Framework: None specified explicitly, but the website appears to use custom frontend development without a specific frontend framework like React, Angular, or Vue.js.
  • Languages: HTML5, CSS3, JavaScript for the structure, styling, and interactivity of the website.
Libraries:

  • Font Awesome: Used for icons throughout the website.
This technology stack ensures a responsive and visually appealing frontend for the department website, leveraging standard web technologies and popular libraries for design consistency and user experience enhancement.

4. Development Tools

IDE (Integrated Development Environment):
  • IDE used: Not explicitly stated from the website analysis, but commonly used IDEs could include Visual Studio Code, JetBrains IntelliJ IDEA, or Eclipse.
We use VS Code for our project.
Reason for choosing IDE:

Choosing an IDE such as Visual Studio Code offers several advantages for developing a website like the Mathematics Department:

  • Versatility: Visual Studio Code supports a wide range of programming languages and frameworks commonly used in web development, including HTML, CSS, JavaScript, and various frontend frameworks like React or Angular if needed.
  • Extensions and Plugins: It provides a rich ecosystem of extensions and plugins that enhance productivity by offering features like syntax highlighting, code completion, debugging support, version control integration, and more.
  • User Interface: Its intuitive user interface and customizable layout make it accessible for developers of varying skill levels to navigate and manage project files efficiently.
  • Community Support: Visual Studio Code benefits from a large and active community, ensuring continuous updates, bug fixes, and support through forums, documentation, and tutorials.
  • Integration with Other Tools: It seamlessly integrates with other tools commonly used in web development workflows, such as Git for version control and various build tools and extensions for task automation.
By using Visual Studio Code or a similar IDE, developers can streamline the development process, maintain code quality, and collaborate effectively on building and maintaining the department website.

5. Version Control: GitHub

Repository Setup:
  • Establishes a repository on GitHub with clear naming and structure.
  • Includes main branches (main or master) and development branches (develop).
  • Readme file provides project details and instructions.
Branching Strategy:
  • Uses main branches for stable releases and development branches for ongoing work.
  • Feature branches are named for specific features or fixes.
  • Release branches prepare code for production deployment.
Collaborative Workflow:
  • Developers create pull requests (PRs) for code review and merging.
  • Code reviews ensure quality and provide feedback.
  • Issues tracked for tasks and bug fixes.
  • Integrates CI for automated testing and deployment.
Communication:

Our team's development process for the Mathematics Department website. As a team of six, each member independently writes and manages their code on separate branches in our GitHub repository. After completing tasks, we push our work to GitHub to maintain transparency and facilitate collaboration through code reviews and discussions. This approach ensures efficient progress and quality assurance in our development efforts.

6. Deployment Strategy

Our deployment strategy for the Mathematics Department website involves using GitHub Pages. GitHub Pages provides a straightforward and efficient platform for hosting static websites directly from our GitHub repository. This approach ensures accessibility and seamless updates for the department's students and stakeholders, leveraging GitHub's reliability and integration capabilities.
